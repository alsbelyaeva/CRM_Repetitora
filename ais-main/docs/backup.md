# Резервное копирование PostgreSQL

В проекте предусмотрен простой локальный механизм резервного копирования БД PostgreSQL для демонстрационной версии ВКР. Он использует стандартную утилиту `pg_dump` и берет параметры подключения только из `DATABASE_URL`.

## Создание backup-файла

Из папки backend:

```bash
npm run db:backup
```

Скрипт создаст папку `backups`, если ее еще нет, и сохранит файл вида:

```text
backups/backup_2026-05-07_18-30-00.sql
```

Если `pg_dump` недоступен в системе, нужно установить PostgreSQL tools и добавить `pg_dump` в `PATH` или выполнить резервное копирование через Docker.

## Команда для Docker

Если проект запущен через `docker compose`, можно создать backup напрямую из контейнера PostgreSQL:

```bash
docker compose exec -T db pg_dump -U ais_user -d ais_db --no-owner --no-privileges > ais-main/backups/backup_2026-05-07_18-30-00.sql
```

В этой команде имя пользователя и БД должны соответствовать значениям из `DATABASE_URL`/`docker-compose.yml`.

## Восстановление из backup-файла

Перед восстановлением убедитесь, что выбрана правильная база данных. Для восстановления можно использовать `psql`:

```bash
psql "$DATABASE_URL" < backups/backup_2026-05-07_18-30-00.sql
```

Для Docker-варианта:

```bash
docker compose exec -T db psql -U ais_user -d ais_db < ais-main/backups/backup_2026-05-07_18-30-00.sql
```

## Хранение файлов

Файлы резервных копий могут содержать персональные данные клиентов, расписание, оплаты и настройки пользователей. Поэтому содержимое папки `backups` не нужно коммитить в git. В репозитории сохраняется только пустой файл `backups/.gitkeep`, чтобы папка была видна в структуре проекта.
