# Система автоматизированного управления взаимоотношениями с клиентами для индивидуальных специалистов

Веб-приложение для автоматизации работы репетитора или другого индивидуального специалиста. Система объединяет учет клиентов, расписание, оплаты, запросы на занятия, отчеты и интеллектуальную поддержку выбора времени занятия.

Ключевая особенность проекта — не просто хранение расписания, а поддержка принятия решений: клиент может предложить несколько временных слотов, а система оценивает каждый вариант и помогает выбрать наиболее подходящий.

## Основные возможности

- регистрация и авторизация пользователей;
- роли `Администратор` и `Преподаватель`;
- управление клиентами, контактами, адресами занятий и статусом VIP;
- просмотр и редактирование расписания;
- создание одиночных и регулярных занятий;
- создание личных событий без привязки к клиенту;
- мягкая отмена занятий и запросов слотов без удаления из базы данных;
- восстановление отмененных занятий и запросов;
- учет оплат и выгрузка отчетов в CSV;
- восстановление пароля по email;
- Telegram-уведомления о занятиях;
- QR-коды и ссылки для подключения Telegram-бота преподавателем и клиентами;
- учет времени дороги между занятиями через 2GIS;
- адаптивная верстка для desktop и мобильных экранов;
- базовое резервное копирование PostgreSQL через `pg_dump`;
- автоматизированные API-тесты, тесты качества и минимальное нагрузочное тестирование серверной части.

## Smart Slotting

Модуль Smart Slotting использует модель взвешенного ранжирования (Weighted Scoring Model).

Формула:

```text
Score(slot) = Σ(criterion_i × weight_i)
```

Критерии ранжирования:

- близость занятия по времени;
- компактность расписания и заполнение окон;
- тип дня: рабочий или нерабочий;
- VIP-приоритет клиента;
- транспортная достижимость между занятиями с учетом 2GIS;
- наличие конфликтов с занятиями или личными событиями.

Вес каждого критерия настраивается пользователем во вкладке настроек. Система показывает не только итоговый балл, но и пояснение по каждому критерию.

## Регулярные занятия

Преподаватель может создать серию занятий, например каждую среду в 17:00. Для серии задаются:

- клиент;
- день недели;
- время начала;
- длительность;
- дата начала;
- дата окончания или количество повторений.

Если отдельная дата серии конфликтует с уже существующим занятием или личным событием, она не исчезает из системы. Такой вариант сохраняется как предложенный слот с меткой коллизии, чтобы преподаватель видел проблемную дату и мог принять решение.

## Личные события

В календаре можно создавать события без клиента: личные дела, экзамены, занятость, встречи и другие блоки времени. Такие события:

- отображаются в расписании вместе с занятиями;
- визуально отличаются от учебных занятий;
- учитываются при проверке конфликтов;
- учитываются в Smart Slotting как занятость преподавателя.

## Telegram-уведомления

Система поддерживает Telegram-бота для напоминаний о занятиях. Уведомления отправляются:

- преподавателю за 24 часа и за 1 час до занятия;
- клиенту, если у клиента подключен Telegram Chat ID.

Если `TELEGRAM_BOT_TOKEN` или Chat ID не указаны, приложение не падает, а пропускает отправку и пишет результат в лог.

Для подключения используются персональные ссылки:

```text
https://t.me/<bot_username>?start=teacher_<userId>
https://t.me/<bot_username>?start=client_<clientId>
```

## Переменные окружения

Реальные `.env`-файлы не добавляются в Git. Для настройки используются шаблоны:

- `ais-main/.env.example`;
- `ais-frontend/.env.example`.

Основные переменные серверной части:

```env
DATABASE_URL=postgresql://ais_user:ais_password@localhost:5432/ais_db?schema=public
JWT_SECRET=change_me
PORT=4000
FRONTEND_URL=http://localhost:5173
CORS_ORIGINS=http://localhost:5173,http://127.0.0.1:5173

SMTP_HOST=
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=
SMTP_PASSWORD=
SMTP_FROM=

TELEGRAM_BOT_TOKEN=
TELEGRAM_BOT_USERNAME=
TWO_GIS_API_KEY=
```

Основная переменная клиентской части:

```env
VITE_API_URL=http://localhost:4000
```

## Запуск через Docker

1. Скопировать шаблоны переменных окружения:

```powershell
copy ais-main\.env.example ais-main\.env
copy ais-frontend\.env.example ais-frontend\.env
```

2. При необходимости указать ключи SMTP, Telegram и 2GIS в `.env`.

3. Запустить проект:

```powershell
docker compose up --build
```

При запуске через Docker backend автоматически применяет Prisma-миграции перед стартом приложения.

После запуска:

- frontend: `http://localhost:5173`;
- backend API: `http://localhost:4000`;
- Swagger/OpenAPI: `http://localhost:4000/docs`;
- PostgreSQL: `localhost:5432`.

## Миграции и тестовые данные

Если проект запускается без Docker, миграции нужно применить вручную:

```powershell
cd ais-main
npx prisma migrate deploy
```

Заполнить базу тестовыми данными:

```powershell
npm run seed
```

В seed создаются учетные записи администратора, преподавателей, клиентов и демонстрационные данные для проверки функций.

## Резервное копирование

Для создания локальной резервной копии PostgreSQL:

```powershell
cd ais-main
npm run db:backup
```

Копия создается через `pg_dump` в папке `ais-main/backups` с именем вида:

```text
backup_YYYY-MM-DD_HH-MM-SS.sql
```

Файлы резервных копий не добавляются в Git. Инструкция по созданию и восстановлению находится в `ais-main/docs/backup.md`.
